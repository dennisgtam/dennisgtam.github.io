Technical Blog 3 Arrays and Hashes

July 27th, 2015

This week I'm going to talk about two very important and different data structures: arrays and hashes. Let's dive right into it.

Arrays

An array is a data structure that can hold multiple values. Unlike other languages, Ruby arrays can be of any size. They can also contain multiple types of objects. An array can contain strings, integers, arrays, and hashes.

Arrays are like ordered lists. You can access the elements of the list with an index. The index starts at zero. Thus, to access the first element of the list we call array_name[0]. The second element would be accessed with array_name[1]. You can also access the last element with array_name[-1], the second to last element with array_name[-2] etc.

The order of the elements is determined when you create the list and when you add or remove elements. Two of the primary methods to add and remove elements from an array are push and pop respectively. Let's take a look at some code.

my_array = ['first', 'second', 'third']
my_array[0] #=> 'first'
my_array[-1] #=> 'third'

my_array.push('fourth') #adds an element to end of the list
my_array[-1] #=> 'fourth'

my_array.pop #removes last element of list
my_array[-1] #=> 'third'

One of the most common applications of lists is to iterate over each element and do something to it. This is best accomplished with a "each do" type of statement. Let's look at a program that will find the largest integer in a list.

def largest_integer(list_of_nums)
  largest = list_of_nums[0]
  list_of_nums.each  do |num|#This is the each do loop. For each "num" in list_of_nums do the following:
    if num > largest
      largest = num
    end
  end
  return largest
end

The method takes in a list of integers. To walk through step by step, the variable largest is set to the first integer in the array. Next, we iterate through the array with a "each do" loop. The loop compares each element of the array with largest. As it goes through, if an element is bigger than largest, largest is assigned that new value. In the end, we get the largest number.

Hashes

Hashes are similar to arrays in that they also store values. However, they store elements in a key value pair. It'll make more sense with a bit of code.

my_hash = { first_key: 'first_value', second_key: 'second_value', third_key: 'third_value'}

There are multiple ways to assign values to a hash but the above is just one way. As you can see, each value is attached to a key. Unlike arrays, my_hash[0] will not do anything. That's because hashes are not ordered and "first element" in a hash doesn't mean anything. It's a bit complicated as to why, but we may get to that in a future post. To access a value, we call it's key. Here's some code.

my_hash[:first_key] #=> 'first_value'
my_hash[:second_key] #=> 'second_value'
my_hash[:third_key] #=> 'third_value'

Hashes are very good for storing descriptive information. A key can tell you what kind of value is being accessed. It's a lot more descriptive than array[3]. Let's have a look.

personal_hash = {first_name: 'Dennis', age: 26, sex: 'male'}
personal_array = ['Dennis', 26, 'male']

From the above, personal_hash[:first_name] and personal_array[0] both return the same element. As does personal_hash[:sex] and personal_array[2]. However, if someone changes the array, the index could be changed so array[2] could point to something else. On the other hand, the keys in always point to a certain value.

There's a lot more to hash but I'm well over my timebox. We'll get back to hashes in more detail in the future. Thanks for reading!
