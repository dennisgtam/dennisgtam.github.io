While we've covered HTML, CSS, and now JS, we haven't really gotten into building any web applications yet. In fact, we haven't built any web application at all yet. A web framework helps us bridge the gap between the Ruby we've been using and the Ruby we'll be using on the web. But first, we have to understand how the web works what a web framework does.

This is a gross oversimplifcation but the internet is a bunch of computers that are connected and know how to talk each other. Files on these computers are most often collected onto web servers. A web server both the physical device and the application that holds all these files. A browser is the principal way that a computer talks to a web server. The browser communicates with the server via something called the HTTP protocol. In response to the HTTP request, the server will a deliver HTML file to the computer that called it and the browser will render the HTML.

Cultural Blog 6: Stereotype Threat

August 10th, 2015

Stereotype threat is the phenomenom in which negative stereotypes of one's self leads to lower performance in various aspects of life. This correlation has shown up in a number of studies. One of the most prominent involed telling students of African background that the test they were about to take was diagnostic of intelligence. African American students who were told the test was non diagnostic of intelligence scored higher than those who were told the test was indicative of intelligence.

I do not believe I've suffered from stereotype threat too recently. The best example I could think of is in sports. In the fall of 2014, I joined a kickball league to have something to do on Sunday afternoons and to make some new friends. While I make sure to exercise regularly, I usually do so by lifting and swimming. I've never been good at sports and as a I kid I was often picked last for most team sports. I went into each kickball game with the mentality that I wouldn't be great at it. We didn't win our season but we weren't at the bottom either. My performance was okay. I'm not sure if I would of improved if I had believed I was better at sports.

Technical Blog 3: Arrays and Hashes

July 27th, 2015

This week I'm going to talk about two very important and different data structures: arrays and hashes. Let's dive right into it.

Arrays

An array is a data structure that can hold multiple values. Unlike other languages, Ruby arrays can be of any size. They can also contain multiple types of objects. An array can contain strings, integers, arrays, and hashes.

Arrays are like ordered lists. You can access the elements of the list with an index. The index starts at zero. Thus, to access the first element of the list we call array_name[0]. The second element would be accessed with array_name[1]. You can also access the last element with array_name[-1], the second to last element with array_name[-2] etc.
