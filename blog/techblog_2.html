Technical Blog 2: Element Display Types!

July 19th, 2015

Today I'm writing about a topic that caused me a lot of grief while I was working through the challenges. I didn't even know the difference between display types. Certain positioning attributes don't work for every display type and it took me awhile to try and figure it out. I know there's a lot of jargon there, but some examples should clear it up.

Every element in a HTML file can be assigned a display type. Usually we do this inside of a css file by writing "display: [TYPE]". There are 3 primary types of displays. Instead of writing many paragraphs of text, I put together a little HTML+CSS demonstration. We'll use some colored div blocks to show what they look like.

I have 3 sets of divs each containing 3 divs. Notice in the css file, that I've set heights and widths for every type of div. They also have different color borders to help differentiate them. When to use each type of display is something I'm still getting used to. Many HTML elements are a certain display type by default. For example, paragraphs and headers are block level elements. Image and links are inline elements.
